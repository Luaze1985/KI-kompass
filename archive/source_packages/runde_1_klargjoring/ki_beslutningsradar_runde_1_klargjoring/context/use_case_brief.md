# Goal

Utvikle et beslutningsrammeverk som hjelper rådgiver/prosjektgruppe/leder med å vurdere om KI skal støtte, automatisere eller ikke brukes i en beslutningsprosess.

# Expected Output

Første nyttige output:

- datafangster i Markdown
- første syntese
- kriterier for målklarhet og separabilitet
- kontrollkrav for forklarbarhet, human oversight og anti-overreliance
- første beslutningsloggstruktur
- Codex/GitHub-handoff

# Constraints

- ikke bygg app ennå
- ikke gjør forklarbarhet til kompassakse
- ikke gjør linser til kompassdimensjoner
- ikke automatiser dømmekraft
- hold filer små og konkrete

# Nearest Existing Profile

`concept_context`

Begrunnelse: Arbeidet starter fra idé, chat, bilder, Notion, Google Drive, GitHub-funn og rammeverk. Målet er først å lage kontekstpakker, datafangster og handoff, ikke kode.

# Reuse Signal

Høy. Hvis modellen testes i flere arbeidsprosesser, kan den senere bli et gjenbrukbart case-mønster eller app-workspace.
