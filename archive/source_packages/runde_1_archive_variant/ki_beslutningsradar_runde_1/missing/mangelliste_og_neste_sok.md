# Mangelliste og neste søk

## Kritiske mangler før samlet syntese

1. **Ledertråd**
   - Må samle prosjektets rød tråd, hovedspørsmål, foreløpige beslutninger og avgrensninger.

2. **Målklarhet**
   - Må definere når et mål er klart nok til at KI kan støtte beslutningen.
   - Må ikke blandes med separabilitet.

3. **Separabilitet**
   - Må definere hvor direkte KI-output kan brukes.
   - Må dekke skjønn, feilkonsekvens, reversibilitet, ansvar, relasjon/tillit, kontekst og rettigheter.

4. **Forklarbarhet + human oversight**
   - Må avklare hva som skal forklares for hvem.
   - Må definere forskjell på reell human oversight og gummistempel.
   - Må kobles til stoppregler og beslutningslogg.

5. **Samlet kildeoversikt**
   - Må samle kilder fra alle datafangster.
   - Må merke primærkilder, forskning, praktiske veiledere og svake kilder.

6. **Testcaser**
   - Trengs for å teste modellen før app/MVP.
   - Foreslåtte caser: HR-sak, servicebeslutning, ressursprioritering, dokumentanalyse, lavrisiko rutineoppgave, høyere risiko individpåvirkning.
