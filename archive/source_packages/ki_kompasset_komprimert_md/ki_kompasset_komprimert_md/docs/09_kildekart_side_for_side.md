# Kildekart side for side

Dette er et kort kart over innholdet i PDF-en, basert på sidevis gjennomgang.

| Side | Innhold | Bruk i prosjektet |
|---:|---|---|
| 1 | Tittel og sammendrag. KI-kompasset presenteres som strategisk diagnoseverktøy. | Brukes i `01_kort_sammendrag.md`. |
| 2 | Introduksjon. Hastighet i KI-adopsjon gir risiko for å løpe i feil retning. | Begrunner kvalitetsport før koding. |
| 3 | Forskningsspørsmål, teoretisk bidrag og beslutningsnivå som analysenivå. | Støtter at appen bør jobbe med beslutningscase. |
| 4 | KI som billigere prediksjon. Prediksjon er ikke beslutning. | Brukes som grunnskille i datamodell. |
| 5 | KI som situert strategisk ressurs. Behov for tydelig verdiforslag. | Brukes til kart + kompass-tankegang. |
| 6 | Metode og analytisk rammeverk. Abduktiv teoretisering og idealtyper. | Viser at modellen er diagnosemodell, ikke fasitmaskin. |
| 7 | Separabilitet: teknisk, legitimitetsbasert og regulatorisk. | Viktig for HR/HMS/persondata. |
| 8 | Målklarhet og figur med fire kvadranter. | Kjernefiguren for UI og logikk. |
| 9 | Kvadrant I, II og III forklares. | Brukes i `04_kvadrantene.md`. |
| 10 | Kvadrant IV, diagnose og feildiagnose langs målklarhet. | Brukes i risiko- og strategi-notat. |
| 11 | Feildiagnose langs separabilitet. Regulatorisk, teknisk og legitimitetsrisiko. | Brukes som automatiseringsstopp. |
| 12 | Konkurransefortrinn og stabil kvadrantposisjon. | Brukes for senere strategivurdering. |
| 13 | Konklusjon. Kompass + kart. Strategidokumentet er kartet. | Brukes som hovedmetafor for appen. |
| 14-16 | Referanser. | Brukes som kildegrunnlag, ikke operativ modell. |

## Viktigste figur

Figuren på side 8 viser aksene:

```text
Målklarhet: lav -> høy
Separabilitet: lav -> høy
```

Og fire roller:

```text
I   Forsterket skjønn
II  Automatisert beslutning
III Utforskende støtte
IV  Strategisk autonomi
```
